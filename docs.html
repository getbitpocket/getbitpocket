1 Configure Bitcoin Address

In order to use BitPocket a Bitcoin Address is required, which is under your control. At first startup of the BitPocket App the Bitcoin Address needs to be set.

BitPocket allows two different configuration options for addresses:

  • Static Address: The configuration can be based on a Static Address. All payments with will always go to the configured Static Address. It will be easy also for others to track all transactions publicly on the blockchain.
  • Master Public Key: In constrast to the Static Address a Master Public Key can be used. Based on an index a new Bitcoin Address can be generated for each payment and your privacy is highly improved. Many wallets allow for exporting the respective Master Public Key (e.g. Electrum, Android Bitcoin Wallet).
2 Additional Configuration Options

Inside the Settings section of the BitPocket App additional configuration options can be found:

  • Merchant Name: Each Bitcoin Payment Request offers a label to display e.g. the merchants name to the payer. The label is encoded into the QR-Code.
  • Currency Format: The underlying currency format of the app. (e.g. 1,000.00 or 1.000,00)
  • Bitcoin Unit: There are different bitcoin units: BTC, mBTC or bits.
  • Currency and Currency Exchange Rate: Currently Blockchain.info and BitcoinAverage are the two integrated providers of exchange rates, which are updated at a 5 minute interval.
  • Static Address: The Static Address can be reset, activated or deactivated.
  • Master Public Key: The Master Public Key can be reset, activated or deactivated. Additionally, an index can be set, at which position the address generation should be started.
3 Request Bitcoin Payments

The main screen of the App allows for the input of the respective payment amount.

After the "request amount" button is clicked, the following is happening:

  • Payment Request: A payment request will be visible and encoded as a QR-Code. The payer can easily scan this code with a respective Bitcoin Wallet.
  • Payment Verification: For approximately 1.5 minutes the payment request will be presented before it will time out. If the payer made the payment withing this timeframe a payment result screen is shown.

4 Transaction History

All transactions will be available within a history list.

  • Each transaction has a icon which is colored either green or yellow. Green means that more than 6 transaction confirmations are available. Yellow means that there are less that 6.
  • Each transaction item within the list can be swipped to the left to open additional options. Either the transaction can be removed from the list (The Bitcoins are untouched), or details can be looked up, within a blockchain explorer.
